# 全局规则

## 沟通与执行原则

- 与用户交流、文本文件（包括 `README.md`）默认用中文；英文术语、代码、脚本名、API 名和配置键保留英文。代码注释和程序日志默认用英文。
- 执行长时间 AI 训练、推理或批量实验时，先回应用户；任务放到后台执行，保留沟通能力，并定期报告进度。
- 长任务启动后必须留下可复查的监控信息：后台命令、PID/任务 ID、日志路径、产物路径、开始时间和建议检查间隔。

## 工具与 Skill 使用

- 涉及长段决策树、实验路线、全局规则或多方案取舍时，优先使用 `grill-me` 逐项达成共识；简单执行任务不强行 grill。
- 项目文档结构、`README.md` / `AGENTS.md` / `CLAUDE.md` / `docs/` 的维护规则由 `neat-freak` 定义；大改动、阶段收尾、文档结构变化或需要交接时调用 `neat-freak`，全局规则不复述细节。
- 长任务需要跨回合或长期跟进时，优先设置 Codex heartbeat / automation 做周期性检查；若无法设置，明确告知限制并给出手动复查命令。
- 普通搜索、最新事实、官方文档查询优先使用 Codex 当前可用的内置 web/search 机制。
- 复杂搜索、批量抓取、多页 crawl 或内置搜索不够用时，使用 Tavily / Firecrawl；两者可能受网络、额度或 API key 影响，失败时换另一个或报告原因。
- 本地 Web app、`localhost`、`127.0.0.1`、文件预览和前端截图/交互验证，优先使用 Browser 插件。
- 需要用户真实 Chrome 登录态、cookies、扩展或既有标签页时，使用 Chrome 插件。
- 需要操作非浏览器桌面 App，或 Browser/Chrome 无法完成的 GUI 操作时，使用 Computer Use 插件。
- 高风险或大范围改动优先使用独立 review：Codex 可用 subagent 时派子 agent；主 agent 负责整合结论，不把子 agent 输出直接当最终事实。小改动可以由主 agent 自查。

## 操作边界

- 当前项目目录默认可读取、创建和修改；项目目录外默认只读，未经用户明确要求不得修改或删除。
- 运行测试时，优先使用项目已有测试结构；若需要新增测试文件，放在项目 `test/` 或既有测试目录下。测试生成物也放在测试目录或 `temp/`。
- 中间文件、草稿和非最终交付物默认保存到项目根 `temp/`。
- 禁止 kill、暂停或抢占不是本次任务创建的进程。
- 可以按需新建虚拟环境，但禁止删除已有环境。

## 软删除

- 不使用硬删除代替软删除。删除项目内文件时，移动到项目根 `trash/`，不要放到被删除文件所在目录的 `trash/`。
- 项目根优先使用当前目录：若当前目录有 `.git`、`AGENTS.md` 或 `CLAUDE.md`，使用当前目录；否则向上查找第一个包含这些标志的目录。`README.md` 不参与项目根判断。大多数情况下当前工作目录就是项目根。
- `trash/` 推荐结构：`recommended_delete/`、`review_required/`、`backup/` 和 `README.md`。
- Agent 本次创建的中间文件、无长期价值 smoke 产物放 `recommended_delete/`，可轻量记录。
- 项目原有文件、用户可能关心的文件、批量清理内容放 `review_required/`，必须在 `trash/README.md` 登记原路径、trash 路径、时间、原因和建议动作。
- 有潜在复用价值的旧实现、旧文档、迁移前快照放 `backup/`，在 `trash/README.md` 标为备份保留或待用户确认。

## Review 和 Git

- 每次较大规模代码、脚本或文档结构改动后，必须做 review 和最小测试/smoke；最终说明覆盖范围和未覆盖风险。
- 正式/长期新项目开始实际创建或修改项目文件前，若不在任何 Git 仓库内，默认初始化本地 Git；项目文档入口按 `neat-freak` 规范处理。临时目录、草稿、一次性 smoke/test 工作区不默认初始化。
- 较大改动完成后检查 `git status`，判断是否值得提交；若值得，先向用户说明提交范围和建议 commit message，再请求确认。
- 用户明确要求 commit，或项目规则明确允许自动提交时，按对应规则执行。不要把用户未确认、任务无关或来源不明的改动混入提交。
