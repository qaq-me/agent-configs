# 全局规则

## 基本规则

- 与用户交流以及项目文档（包括`docs/`下的文档、`README.md`、`CLAUDE.md`、`AGENTS.md`）默认用中文，但英文术语保留，可附带中文解释；非文档内容如代码注释、程序日志等默认用英文。
- 在执行任务前，先给用户回应，然后再执行，执行过程中也应让用户知晓当前状态。
- 遇到信息量大、决策点多或方案分支复杂的任务，不要一次性给用户大量信息，先梳理需要决策的问题，使用 grill-me skill 与用户达成共识。
- 需要网络搜索、抓取时，先使用内置网络搜索抓取工具；如果内置网络搜索抓取工具不足以完成任务，使用 Tavily / Firecrawl 的 search、extract/scrape、crawl 相关 skill；如果以上方式失败，或需要实际访问网站、登录态、页面交互时，使用 Chrome 插件访问真实网站。

## 操作边界

- 项目根目录外默认只读，未经用户要求不得修改删除。
- 禁止 kill 不是在本次任务创建的进程。
- 允许按需新建虚拟环境，禁止删除已有环境。

## 项目管理

- 在实际创建或修改文件前，若当前工作目录不在任何 git 仓库内，默认将当前工作目录初始化为本地 git 仓库，并用 git-ignore skill 配置 .gitignore。
- 每次较大改动或完成阶段任务后：
  - 先做 review 和必要测试，高风险或大范围改动使用 subagent 进行独立 review，小范围改动可以自查。
  - review 和测试通过后，使用 docs-manager skill 更新对应本次改动需要更新的文档。
  - 文档更新后检查 `git status`，判断是否值得提交，并向用户报告。首先报告本次的任务完成情况、review 和测试情况以及文档更新情况，若值得提交，再加上提交范围和 commit message 说明，请求确认，如果存在远端仓库，同时向用户确认是否推送到远端。

## 文件管理

- 根目录默认为 .git 所在目录，若当前目录非根目录则向上寻找包含 .git 的目录。
- 中间文件、草稿和非最终交付物默认保存到根目录下 `temp/`（如果没有则创建）。
- 进行测试时，测试文件及生成物全部放在根目录下 `test/`（如果没有则创建）。
- 删除默认为软删除，将需要删除的文件或目录移动到根目录下 `trash/`（如果没有则创建），注意不是被删除文件所在目录下 `trash/`。
